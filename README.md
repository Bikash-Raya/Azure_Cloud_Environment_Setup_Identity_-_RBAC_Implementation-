## 📋 Overview

This repository documents the setup and administration of a **Microsoft Azure cloud environment** focused on infrastructure provisioning, identity management, automation, and access control.

The project includes deployment of a Windows Server 2016 virtual machine, configuration of users and security groups using Microsoft Entra ID, implementation of dynamic group membership rules, and Role-Based Access Control (RBAC) using custom roles.

This lab simulates real-world enterprise cloud administration practices.

## 🧪 Implementation Summary

### ☁️ Azure Infrastructure Setup
- Created resource group **MoonTechRG** in Australia East region  
- Deployed **Windows Server 2016 VM (MoonTechVM)**  
- Configured networking and storage resources  

---

### 👤 Identity & Group Management
- Created users:
  - Tim  
  - Jack  
  - Ronaldo  
  - Messi  
- Created security groups:
  - MoonTechSales  
  - MoonTechMarketing  

---

### ⚙️ Dynamic Group Configuration
- Assigned department attributes (Sales / Marketing)  
- Configured automatic group membership rules  
- Eliminated manual user assignment process  

---

### 🔐 Role-Based Access Control (RBAC)
- Created custom roles:
  - SalesAdmin  
  - MarketingAdmin  
- Assigned roles to users:
  - Ronaldo → SalesAdmin  
  - Messi → MarketingAdmin  
- Implemented delegated administration model  

---

### 📊 Validation
- Verified automatic group assignment  
- Confirmed RBAC role permissions  
- Tested identity and access configurations
